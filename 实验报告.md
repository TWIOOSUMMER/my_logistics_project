## 二、实验原理

### 2.1 Django 框架架构

Django 是一个基于 Python 的高级 Web 框架，遵循 MVT（Model-View-Template）设计模式：

- **Model（模型层）**：负责与数据库交互，定义数据结构。通过 ORM（对象关系映射）机制，开发者可以使用 Python 类来定义数据表结构，Django 会自动生成对应的 SQL 语句完成数据库操作。
- **View（视图层）**：负责处理业务逻辑，接收 HTTP 请求，从模型层获取数据，并将数据传递给模板层进行渲染。
- **Template（模板层）**：负责数据的展示，使用 Django 模板语言将动态数据嵌入 HTML 页面中。

### 2.2 Django ORM 原理

Django ORM 提供了一套面向对象的数据库操作接口：
- 每一个 Django Model 类对应数据库中的一张表
- Model 类的属性对应表中的字段
- 通过 `objects` 管理器进行数据库的 CRUD 操作：
  - `Model.objects.all()` — 查询所有记录
  - `Model.objects.create(**kwargs)` — 创建新记录
  - `Model.objects.filter(**kwargs)` — 按条件筛选记录
  - `model_instance.delete()` — 删除记录

### 5.1 数据模型（models.py）

```python
from django.db import models


class GoodsTb(models.Model):
    """货物信息表"""
    goods_name = models.CharField(max_length=50, verbose_name='货物名称')
    category = models.CharField(max_length=30, verbose_name='货物类别')
    weight = models.DecimalField(max_digits=10, decimal_places=2, verbose_name='重量(kg)')
    price = models.DecimalField(max_digits=10, decimal_places=2, verbose_name='单价(元)')
    stock = models.IntegerField(default=0, verbose_name='库存数量')

    class Meta:
        app_label = 'logistics'
        db_table = 'goods_tb'
        verbose_name = '货物信息'
        verbose_name_plural = '货物信息'

    def __str__(self):
        return self.goods_name


class OrderTb(models.Model):
    """订单信息表"""
    ORDER_STATUS = [
        ('pending', '待处理'),
        ('shipping', '运输中'),
        ('delivered', '已签收'),
        ('cancelled', '已取消'),
    ]

    order_number = models.CharField(max_length=30, unique=True, verbose_name='订单编号')
    customer_name = models.CharField(max_length=30, verbose_name='客户名称')
    goods_name = models.CharField(max_length=50, verbose_name='货物名称')
    quantity = models.IntegerField(default=1, verbose_name='数量')
    status = models.CharField(max_length=20, choices=ORDER_STATUS, default='pending', verbose_name='订单状态')
    created_at = models.DateTimeField(auto_now_add=True, verbose_name='创建时间')

    class Meta:
        app_label = 'logistics'
        db_table = 'order_tb'
        verbose_name = '订单信息'
        verbose_name_plural = '订单信息'

    def __str__(self):
        return self.order_number
```

### 5.2 视图函数（views.py）

```python
from django.shortcuts import render, redirect, get_object_or_404
from .models import GoodsTb, OrderTb


def index(request):
    """首页：展示所有货物和订单列表"""
    goods_list = GoodsTb.objects.all()
    order_list = OrderTb.objects.all()
    context = {
        'goods_list': goods_list,
        'order_list': order_list,
    }
    return render(request, 'logistics/index.html', context)


def goods_add(request):
    """添加货物：GET显示表单，POST创建记录"""
    if request.method == 'POST':
        GoodsTb.objects.create(
            goods_name=request.POST.get('goods_name', ''),
            category=request.POST.get('category', ''),
            weight=request.POST.get('weight', 0),
            price=request.POST.get('price', 0),
            stock=request.POST.get('stock', 0),
        )
        return redirect('index')
    return render(request, 'logistics/goods_add.html')


def goods_delete(request, goods_id):
    """删除指定货物"""
    goods = get_object_or_404(GoodsTb, id=goods_id)
    goods.delete()
    return redirect('index')


def order_add(request):
    """添加订单：GET显示表单，POST创建记录"""
    if request.method == 'POST':
        OrderTb.objects.create(
            order_number=request.POST.get('order_number', ''),
            customer_name=request.POST.get('customer_name', ''),
            goods_name=request.POST.get('goods_name', ''),
            quantity=request.POST.get('quantity', 1),
            status=request.POST.get('status', 'pending'),
        )
        return redirect('index')
    return render(request, 'logistics/order_add.html')


def order_delete(request, order_id):
    """删除指定订单"""
    order = get_object_or_404(OrderTb, id=order_id)
    order.delete()
    return redirect('index')


def goods_search(request):
    """按货物名称模糊搜索"""
    query = request.GET.get('q', '')
    goods_list = GoodsTb.objects.filter(goods_name__icontains=query) if query else GoodsTb.objects.all()
    order_list = OrderTb.objects.all()
    context = {
        'goods_list': goods_list,
        'order_list': order_list,
        'search_query': query,
    }
    return render(request, 'logistics/index.html', context)


def order_search(request):
    """按订单编号模糊搜索"""
    query = request.GET.get('q', '')
    goods_list = GoodsTb.objects.all()
    order_list = OrderTb.objects.filter(order_number__icontains=query) if query else OrderTb.objects.all()
    context = {
        'goods_list': goods_list,
        'order_list': order_list,
        'order_search_query': query,
    }
    return render(request, 'logistics/index.html', context)
```

## 六、实验心得总结

通过本次实验，我对基于 Django 框架开发 Web 应用有了系统性的认识和实践。

首先，我对 Django 的 MVT 架构模式有了更深入的理解。在开发过程中，数据模型（Model）承担了与 MySQL 数据库交互的核心职责，通过在 Python 代码中定义模型类，Django 的 ORM 机制自动完成了数据库表的创建和 SQL 操作，大大简化了数据库开发工作。这种面向对象的数据库操作方式，让开发者可以专注于业务逻辑而无需编写大量原始 SQL 语句。

其次，视图函数（View）作为连接模型和模板的桥梁，起到了承上启下的作用。通过 request 对象接收前端数据，调用模型层进行数据库操作，再将结果传递给模板渲染，这个数据流转过程的设计清晰且高效。我在实现货物和订单的增删查功能时，采用了 Python 装饰器和重定向机制，使得代码结构紧凑、逻辑清晰。

再次，Django 模板语言的使用让前后端分离开发变得可行。模板中的循环、条件判断等语法简洁直观，配合 CSS 样式可以快速搭建出美观实用的用户界面。本次实验中为页面添加了确认删除的 JavaScript 提示功能，有效防止了用户的误操作。

在环境配置方面，MySQL 数据库的安装和 PyMySQL 兼容层的配置是本次实验的关键环节。通过在 settings.py 中使用 `pymysql.install_as_MySQLdb()` 方法，实现了 PyMySQL 对 MySQLdb 的兼容替代，使得 Django 6.0 能够顺利连接 MySQL 8.0 数据库。

此外，通过编写自动化测试脚本对系统进行验证，我体会到测试驱动开发的重要性。系统的测试用例覆盖了所有功能点，及时发现并修复了 ALLOWED_HOSTS 配置等问题，保证了代码的正确性和可靠性。

在实验中遇到的主要问题包括：MySQL 数据库连接认证失败（需确认正确的用户名和密码）、Django 6.0 版本中测试客户端对 ALLOWED_HOSTS 的严格要求等。通过查阅官方文档和调试日志，这些问题都得到了妥善解决。

总体来说，本次实验让我掌握了使用 Django 框架和 MySQL 数据库构建 Web 信息系统的完整流程，为今后开发更复杂的物流管理平台打下了坚实基础。
